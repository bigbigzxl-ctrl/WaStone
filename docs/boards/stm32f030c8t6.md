# STM32F030C8T6

**MCU:** STM32F030C8T6 — Cortex-M0, 48 MHz max, 64 KB flash, 8 KB RAM, 48-pin LQFP  
**Family:** STM32F0 (`STM32F030x8`)  
**Status:** canonical golden suite verified, DAPLink UART side-packs verified  
**Verification date:** 2026-04-03

---

## Canonical Suite

Suite name: `stm32f030c8t6_golden`  
Pack: `packs/stm32f030c8t6_golden.json`  
Result: **24 / 24 PASS** on the canonical local DAPLink bench

Canonical UART mode:

- `PA9 <-> PA10`
- local DUT loopback
- used by the canonical Golden Suite

This remains the default suite because it is the simplest stable bench
contract and does not depend on host-driven UART traffic.

---

## UART Side-Packs

These are validated side-packs for the alternate DAPLink UART bridge wiring:

- `PA9 -> DAPLink RX`
- `DAPLink TX -> PA10`

### 1. TX-Observed UART Pack

Suite name: `stm32f030c8t6_daplink_uart_observed`  
Pack: `packs/stm32f030c8t6_daplink_uart_observed.json`  
Result: **3 / 3 PASS**

Coverage:

- host-observed UART TX probe
- host-observed multibyte TX
- host-observed banner TX

Use this when you only want to prove the DUT TX path through DAPLink.

### 2. Bidirectional UART Pack

Suite name: `stm32f030c8t6_daplink_uart_bidirectional`  
Pack: `packs/stm32f030c8t6_daplink_uart_bidirectional.json`  
Result: **5 / 5 PASS**

Coverage:

- TX probe
- multibyte TX
- banner TX
- host-driven RX probe
- host-driven echo / request-response

Use this when you want the strongest UART proof on the DAPLink bridge wiring.

## Bench Wiring

### Canonical Golden Suite

| DUT pin | Connection | Role |
|---|---|---|
| SWDIO / SWCLK | DAPLink | SWD debug / flash |
| NRST | DAPLink NRST | reset |
| GND | probe GND | common ground |
| PC13 | onboard LED | visible board-life path |
| PA0 | PA1 | ADC / EXTI |
| PB0 | PB1 | GPIO / timer |
| PB8 | PB9 | capture / PWM |
| PA7 | PA6 | SPI1 MOSI to MISO |
